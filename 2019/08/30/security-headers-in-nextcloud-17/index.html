Security headers in Nextcloud 17

The third beta on Nextcloud 17 has just been released and that is as good a time as any to update you on the Nextcloud security header changes that are coming to you in Nextcloud 17.

Content Security Policy

The form-action directive has been added to the CSP. By default this will only allow forms to post to ‘self’. This means that all form submissions by default have to be submited to the origin.

‘unsafe-eval’ has been disabled by default for some time now. But please do note that the ability for apps to overwrite this will disappear in a future version of Nextcloud. So if you have not already, fix your code!

Feature Policy

With Nextcloud 17 a new shiny security header is shipper. The feature policy header tells your browser what features it can use.
